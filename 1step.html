QA Monitor — 시작 가이드

Linear × Slack QA 자동화 설정

이 가이드를 따라 설정하면 매일 지정된 시각에 QA 현황 리포트가 자동으로 발송됩니다.

Setup
설치 및 초기 설정
순서대로 진행하면 30분 안에 완료됩니다.
1

패키지 설치

# 가상환경 생성 (권장) python -m venv .venv source .venv/bin/activate # Windows: .venv\Scripts\activate # 의존성 설치 pip install -r requirements.txt
2

API 키 설정 (.env)

# .env.example 복사 후 편집 cp .env.example .env
변수설명획득 방법
LINEAR_API_KEY Linear Personal API Key Linear → Settings → API → Personal API keys
SLACK_BOT_TOKEN Slack Bot OAuth Token api.slack.com/apps → OAuth & Permissions → Bot Token
SLACK_CHANNEL_ID 발송할 채널 ID 채널 우클릭 → "채널 ID 복사"
3

Slack 앱 권한 설정

api.slack.com/apps 에서 새 앱을 만들고 아래 Bot Token Scopes를 추가하세요.

chat:write im:write users:read users:read.email
💡 앱 설치 후 채널에 초대해야 메시지를 보낼 수 있습니다 → /invite @앱이름
4

config.yaml 설정 — 핵심 설정 파일

⚙️
config.yaml
이 파일 하나에서 프로젝트명, Linear 연동, Slack 채널, 스케줄, 배포 기준을 모두 설정합니다.
project: name: "My Service QA" # 대시보드 제목 release_date: "2026-04-15" # D-Day 계산용 릴리즈 목표일 linear: team_id: "abc123" # Linear Team ID (필수) project_id: "" # 프로젝트 단위 모니터링 시 cycle_id: "" # 스프린트 단위 모니터링 시 qa_labels: ["QA", "Test Case"] # Linear QA 카드 라벨명 bug_labels: ["Bug", "Defect"] # Linear 버그 라벨명 slack: summary_channel: "C0XXXXXXXXX" send_dm_to_assignees: true # 담당자별 DM 발송 여부 user_map: # Linear 이름 → Slack User ID "홍길동": "U0XXXXXXXXX" "Jane Doe": "U0YYYYYYYYY" scheduler: send_time: "09:00" # 매일 발송 시각 (HH:MM) timezone: "Asia/Seoul" dashboard: output_dir: "output" checklist_path: "deployment_checklist.md" # ← 배포 체크리스트 항목 파일 max_bugs_display: 50 # 대시보드 버그 목록 최대 표시 수 trend_days: 14 # 이슈 추이 차트 기간 (일) exit_criteria: # ← 수치 기준 여기에 입력 (자동 Pass/Fail) urgent_bug_max: 0 high_bug_max: 0 medium_bug_max: 5 test_completion_min_pct: 100 test_pass_min_pct: 95
🔍 Linear ID 확인: team_id / project_id / cycle_id는 해당 페이지 URL 끝부분에서 확인할 수 있습니다.

Exit Criteria
QA 종료 기준 & 배포 기준
수치 기준은 config.yaml → exit_criteria 에 입력 / 배포 체크리스트 항목은 src/dashboard_generator.py:313 에서 직접 수정

QA 종료 기준 (Exit Criteria)

아래 항목을 모두 충족해야 QA가 종료됩니다.
필수 Urgent(P0) 버그 0건
필수 High(P1) 버그 0건
필수 테스트 케이스 실행률 100%
필수 핵심 기능 회귀 테스트 100% 통과
권장 Medium(P2) 버그 5건 이하*
권장 테스트 통과율 95% 이상
권장 재오픈 이슈 0건
선택 Low(P3) 버그 다음 릴리즈 이관
* Medium 이상 Known Issue 예외 처리 시: QA 리더 + PM 공동 승인 필수

배포 기준 (Deployment Criteria)

QA 종료 후 배포 전 최종 확인 항목입니다.
  • QA 종료 기준 전 항목 충족 확인
  • 스테이징 환경 최종 스모크 테스트 통과
  • 성능 테스트 — 응답시간 목표값 이내 (예: p95 < 500ms)
  • 보안 취약점 스캔 이상 없음
  • 데이터 마이그레이션 검증 완료 (해당 시)
  • 롤백 플랜 문서화 및 담당자 숙지
  • 모니터링 / 알림 설정 확인
  • QA 리더 최종 Sign-off
  • PM 배포 승인

FAQ
자주 발생하는 문제
Linear API 오류 (401 Unauthorized)
LINEAR_API_KEY가 올바른지 확인하세요. lin_api_로 시작해야 합니다.
Slack not_in_channel 오류
Slack 앱을 채널에 초대하세요: /invite @앱이름
담당자 DM이 안 보내짐
config.yaml의 slack.user_map에 Linear 이름 → Slack User ID 매핑을 추가하세요. Slack에서 프로필 클릭 → "멤버 ID 복사"로 확인 가능합니다.
이슈가 0개로 조회됨
config.yaml의 qa_labels, bug_labels가 Linear 실제 라벨명과 대소문자까지 정확히 일치하는지 확인하세요.
